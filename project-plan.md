# План разработки проекта "Приглашение на свадьбу"

## 1. Цель проекта

Разработать одностраничный сайт-приглашение на свадьбу с красивым минималистичным лендингом, формой подтверждения присутствия, опросом гостей, админ-панелью для управления контентом и списком гостей, сервером, базой данных и Docker Compose для простого запуска.

Лендинг и админ-панель должны работать на одном сайте:

- публичная часть: `/`
- админ-панель: `/admin`
- вход в админ-панель: по ключ-коду

Основные цвета из `instruction.md`:

- фон: `#FDFBF8`
- текст и акценты: `#AD6766`

## 4. Основные сущности базы данных

### Invitation

Приглашение, созданное в админ-панели. По нему генерируется публичная ссылка для гостя или семьи.

Поля:

- `id`
- `slug`
- `displayName`
- `guestType`
- `isActive`
- `createdAt`
- `updatedAt`

Пояснения:

- `slug` - случайная строка для ссылки, например `/invite/k7s9ad2q`
- `displayName` - имя или имена для обращения, например `Светлана и Андрей`
- `guestType` - тип приглашения, например `single`, `couple`, `family`
- `isActive` - возможность отключить ссылку без удаления данных

### Guest

Гость, подтвердивший участие.

Поля:

- `id`
- `invitationId`
- `firstName`
- `lastName`
- `phone`
- `foodPreferences`
- `alcoholPreferences`
- `hasChild`
- `comment`
- `moneyGiftEnabled`
- `moneyGiftAmount`
- `createdAt`
- `updatedAt`

### SiteContent

Контент лендинга, который можно менять из админ-панели.

Поля:

- `id`
- `key`
- `value`
- `type`
- `updatedAt`

Примеры ключей:

- `hero.title`
- `intro.guestNameTemplate`
- `intro.date`
- `intro.place`
- `story.title`
- `story.items`
- `gift.title`
- `gift.description`
- `schedule.items`

### MediaAsset

Фотографии и изображения сайта.

Поля:

- `id`
- `key`
- `url`
- `alt`
- `createdAt`
- `updatedAt`

Изображения хранятся локально на сервере. Для Docker Compose нужно добавить volume, чтобы загруженные фотографии не пропадали после перезапуска контейнера.

### AdminSession или AdminToken

Если вход будет по простому ключ-коду, можно не заводить полноценную модель пользователей. Сервер проверяет ключ из `.env`, выдает короткоживущий JWT, а админские запросы проверяют этот токен.

## 5. API

### Публичные запросы

```http
GET /api/public/content
```

Возвращает весь публичный контент лендинга.

```http
GET /api/public/invitations/:slug
```

Возвращает данные приглашения по случайной строке.

Пример ответа:

```json
{
  "slug": "k7s9ad2q",
  "displayName": "Светлана и Андрей",
  "guestType": "couple"
}
```

```http
POST /api/public/rsvp
```

Создает или обновляет подтверждение гостя. Повторная отправка разрешена, чтобы гость мог исправить данные.

Тело запроса:

```json
{
  "invitationSlug": "k7s9ad2q",
  "firstName": "Имя",
  "lastName": "Фамилия",
  "phone": "+7...",
  "foodPreferences": ["не ем мясо", "не ем рыбу"],
  "alcoholPreferences": ["Красное вино", "Шампанское"],
  "hasChild": false,
  "comment": "Комментарий",
  "moneyGiftEnabled": true,
  "moneyGiftAmount": 5000
}
```

### Админские запросы

```http
POST /api/admin/login
```

Проверяет ключ-код и возвращает токен.

```http
GET /api/admin/content
PUT /api/admin/content
```

Получение и обновление текстов, описаний и настроек лендинга.

```http
GET /api/admin/invitations
POST /api/admin/invitations
PUT /api/admin/invitations/:id
DELETE /api/admin/invitations/:id
```

Управление приглашениями и генерация индивидуальных ссылок.

При создании приглашения админ указывает отображаемое имя, например `Светлана и Андрей`, а сервер генерирует случайный `slug`.

```http
POST /api/admin/media
DELETE /api/admin/media/:id
```

Загрузка и удаление изображений.

```http
GET /api/admin/guests
```

Список гостей.

```http
GET /api/admin/guests/export.xlsx
```

Выгрузка гостей в Excel.

## 6. Лендинг

### Блок 1. Закрытое письмо

Состояние первого экрана:

- визуал закрытого письма
- надпись: `Приглашение на свадьбу`
- легкая анимация появления
- по клику или скроллу письмо раскрывает следующий блок

### Блок 2. Персональное приглашение

Контент:

- `Дорогой/дорогая "Имя"...`
- дата: `пятница, 21 августа 2026`
- место: `Рестобар "Кубик", г. Якутск, ул. Аржакова, 10`
- таймер до свадьбы

Имя гостя берется из приглашения. Админ создает приглашение в админ-панели, сервер генерирует случайную строку, а публичная ссылка выглядит так:

```text
/invite/k7s9ad2q
```

При заходе на страницу клиент отправляет запрос по `slug` и получает `displayName`. Для семейной пары используется одно приглашение с общим отображаемым именем, например `Светлана и Андрей`.

### Блок 3. Общее приглашение и фото

Контент:

- текст из `instruction.md`
- совместная фотография
- адаптивное расположение фото и текста

### Блок 4. Наша история

Контент:

- несколько текстовых блоков
- несколько фотографий
- плавное появление элементов при прокрутке

### Блок 5. Меню

Контент:

- изображение с вилкой, тарелкой и ножом
- текст о предпочтениях и диетических ограничениях
- переход к опросу после подтверждения

### Блок 6. Пожелания по подаркам

Контент:

- основной текст о присутствии как главном подарке
- маленькая подпись: `Конверты приветствуются`
- изображение конверта

### Блок 7. Подтверждение

Контент:

- календарь августа 2026 года
- дата 21 августа визуально обведена
- кнопка `Подтвердить`

Поведение:

- до нажатия на кнопку блок вопросов скрыт
- после нажатия открывается блок вопросов

### Блок 8. Вопросы

Поля:

- предпочтения по еде:
  - нет
  - не ем мясо
  - не ем рыбу
  - вегетарианец
- алкоголь:
  - Красное вино
  - Белое вино
  - Шампанское
  - Виски/коньяк
  - Водка
  - Не буду пить алкоголь
- будет ли ребенок:
  - да
  - нет
- комментарий
- чекбокс `Денежный подарок`
- сумма денежного подарка, если чекбокс включен

Поведение полей:

- предпочтения по еде можно выбрать в любом количестве
- алкоголь можно выбрать в любом количестве
- сумма денежного подарка обязательна, если включен чекбокс `Денежный подарок`

После нажатия `Отправить`:

- открыть модальное окно с полями:
  - имя
  - фамилия
  - номер телефона
- после подтверждения отправить данные на сервер
- если выбран денежный подарок, после успешной отправки показать реквизиты для перевода
- если денежный подарок не выбран, показать благодарственное сообщение
- повторная отправка формы разрешена и обновляет ответ по текущему приглашению

### Блок 9. Свадебное расписание

Визуал:

- линия, похожая на синусоиду
- сердечко двигается по линии при прокрутке
- рядом с событиями векторные изображения

События:

- `13:30` Торжественная регистрация
- `15:00` Фотосессия
- `16:00` Праздничный банкет
- `23:00` Окончание праздничного дня

### Блок 10. Карта

Контент:

- Яндекс Карта
- адрес: `Рестобар "Кубик", г. Якутск, ул. Аржакова, 10`

Карту вставляем через iframe. Полноценная интеграция с API Яндекс Карт не нужна.

## 9. Принятые решения

1. Имя приглашенных задается вручную в админ-панели.
2. Админ создает приглашение, сервер генерирует случайную строку, а по ней формируется ссылка.
3. Для семейной пары используется одно приглашение с общим отображаемым именем, например `Светлана и Андрей`.
4. Реквизиты для денежного подарка будут добавлены позже.
5. Если выбран `Денежный подарок`, сумма обязательна.
6. Алкоголь можно выбрать в любом количестве.
7. Предпочтения по еде можно выбрать в любом количестве.
8. Повторная отправка формы разрешена.
9. Защита от спама на первом этапе не нужна.
10. Фотографии хранятся локально на сервере.
11. Для Яндекс Карты достаточно iframe.
12. Нужна только одна светлая тема.
13. Мультиязычность не нужна.

## 10. Рекомендации

1. Начать с минимального рабочего контура: база, сервер, публичный API, простой лендинг и сохранение RSVP.
2. Админку делать после того, как будет понятно, какие данные реально нужны лендингу.
3. Сложные анимации добавить после готовности структуры, иначе есть риск потратить время на визуал до того, как заработает основной сценарий.
4. Контент хранить в базе сразу, даже если на первом этапе кажется проще зашить его в код. Это упростит админ-панель.
5. Для входа в админку ключ-код + JWT достаточно для небольшого частного сайта. Полноценная система пользователей здесь выглядит избыточной.
